Linux文件权限管理

Posted on 2018-12-05 | | 浏览 次
Words count in article: | Reading time ≈

权限身份

  • 拥有者（Owner）
  • 用户组（Group）
  • 其他（Others）

文件权限类型

  • read:可读
  • write:可以编辑、新增或者修改该文件的内容（不包括删除）
  • execute：可执行

目录权限

  • r:(read contens in directory)读取目录结构列表的权限，可以用ls这个命令将该目录的内容列表显示出来
  • w:(modify contens in directory)
    • 建立新的文件与目录
    • 删除已经存在的文件或目录（不论该文件的权限是什么）
    • 修改文件或目录名字
    • 移动该目录内的文件、目录位置
  • x:(access directory)用户能否进入到该目录成为工作目录，即能否使用cd命令进入到该目录下
  • Note:

修改文件权限

  • chgrp（change group）：修改文件所属用户组
    • chgrp [R] dirname/filename
  • chown(change owner)：修改文件拥有者
    • chown [R] 账号名称 文件或目录
  • chmod：修改文件的权限
    • 数字类型修改文件权限
      • r:4
      • w:2
      • x:1
      • eg: chmod 754 test 修改test文件的权限为rwxr-xr–
    • 符号类型修改文件权限
关键字 身份 操作 权限 权限
chmod u（user） +(加入) r 文件或目录
g(group) -（移除） w
o(others) =（设置） x
o( all) =（设置） x
  • 设置test的权限为-rwxr-xr-x
  • chmod u=rwx,go=rx test
  • chmod a+w test
