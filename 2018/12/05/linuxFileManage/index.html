Linux文件权限管理

Posted on 2018-12-05 | | 浏览 次
Words count in article: | Reading time ≈

权限身份

  • 拥有者（Owner）
  • 用户组（Group）
    • 用户组的存在是为了帮助进行团队协作的
  • 其他（Others）

权限设置重要性

  • 系统保护的功能
  • 团队开发软件或数据共享的功能
  • 避免权限设置不妥的危害

文件属性

文件权限

  • r:可读
  • w:可以编辑、新增或者修改该文件的内容（不包括删除）
  • e:可执行

目录权限

  • r:(read contens in directory)读取目录结构列表的权限，可以用ls这个命令将该目录的内容列表显示出来
  • w:(modify contens in directory)
    • 建立新的文件与目录
    • 删除已经存在的文件或目录（不论该文件的权限是什么）
    • 修改文件或目录名字
    • 移动该目录内的文件、目录位置
  • x:(access directory)用户能否进入到该目录成为工作目录，即能否使用cd命令进入到该目录下
  • Note:
    • 文件和目录的权限区别可以用一个文件（file）和文件盒（directory）来类比;
    • 要开放目录给任何人浏览的时候，应该至少赋予r和权限，但w权限不能随便赋予；
    • 能否读取某个文件的内容，跟该文件所在的目录也有关系（得先有key打开文件盒才能查看文件）

修改文件权限

  • chgrp（change group）：修改文件所属用户组
    • chgrp [R] dirname/filename
  • chown(change owner)：修改文件拥有者
    • chown [R] 账号名称 文件或目录
  • chmod：修改文件的权限
    • 数字类型修改文件权限
      • r:4
      • w:2
      • x:1
      • eg: chmod 754 test 修改test文件的权限为rwxr-xr–
    • 符号类型修改文件权限
      chomodu（owner）+(add)rfilename/diretory name
      g（group）-(sub)w
      o（other）=(set)x
      a（all）
  • 设置test的权限为-rwxr-xr-x: chmod u=rwx,go=rx test
  • chmod a+w test

相对路径和绝对路径

  • 绝对路径：由根目录（／）开始写起的文件名或者目录；
  • 相对路径：相对于目前路径的文件名写法；
    • .:代表当前的目录；
    • ..:代表上一层目录；
    • -:代表前一个工作目录；
    • ～:代表目前使用者身份所在的家目录

隐藏文件

  • 文件名前面多了一个[.]，则代表该文件是隐藏文件或目录；
  • 使用ls -a命令可以查看隐藏文件；

Linux文件目录配置的依据——FHS

  • FHS定义的三层文件目录结构
    • ／（root，根目录）：与系统启动有关；
    • ／usr（unix software resource）:与软件安装／执行有关
    • ／var（variable）：与系统运行过程有关
