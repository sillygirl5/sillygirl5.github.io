Linux文件与目录管理

Posted on 2018-12-06 | | 浏览 次
Words count in article: | Reading time ≈

常用的处理目录的命令

  • cd：切换目录
  • pwd：显示当前目录
  • mkdir：建立一个新的目录
    • mkdir [mp] directory name
    • [m]：设置文件的权限
    • [p]：递归创建所有目录
  • rmdir：删除一个空的目录
    • [p]：连同上一级“空的”目录也一起删除

文件与目录管理

  • 文件与目录的查看：ls [-aAdfFhilnrRSt] 文件或目录名称
    • [a]：全部文件，包括隐藏的文件
    • [d]：仅列出目录本身，而不是列出目录内的文件数据
    • [l]：详细信息
  • 复制（cp）、删除（rm）与移动（mv）
  • cp [-adfilprsu] source destiation
    • [i]：若目标文件已经存在，在覆盖时先询问
    • [p]：连同文件的属性（权限，用户，时间）一起复制过去
    • [r]：递归复制
    • [s]:复制称为符号链接文件（symbolic link），亦称为快捷方式
Note:复制时需要考虑：
  • 是否需要完整的保留源文件信息？
  • 源文件是否为符号链接文件？
  • 源文件是否为特殊文件，例如FIFO、socket等？
  • 源文件是否为目录？
  • rm [-fir] file/dir
    • [f]：force,忽略不存在的文件，不会出现警告
    • i：交互模式，在删除是会提醒
    • r：递归删除
  • mv [-fiu] source destination
    • f: force,如果目标文件已经存在，不会询问直接覆盖
    • i：如果目标文件已经存在，会询问覆盖
    • u：如果目标文件已经存在，且source比较新，才会更新（update）

文件内容的查看

  • cat [AbEnTv] filename/dirname
    • b：打印出非空白行的行号
    • n：打印出所有行的的行号
  • tac:从最后一行开始显示，cat倒着写
  • nl [bnw] file：添加行号打印
  • 可翻页查看
    • more
      • 空格键（space）：向下翻一页
      • Enter：向下一行
      • ／字符串：向下查找字符串这个关键字
      • b／ctrl+b：往回翻页
      • :f：立刻显示出文件名及目前显示的行数
    • less
      • pagedown：向下翻动一页
      • pageup：向上翻动一页
      • ？字符串：向上查找字符串
      • g：前进到这个数据的第一行
      • G：前进到这个数据的最后一行
  • 数据截取
    • head [-n number] filename :取出数据的前几行；
    • tail [-n number] filename:取出文件的后门几行；
  • 查看莫文件的第五行到第十行数据
  • head -n 10 1.txt|tail -n 5;
  • (|):表示前面的命令所输出的信息，通过管道交由后续的命令继续使用
  • 非纯文件文件：od

修改文件的时间或者创建新文件：touch

  • Linux涉及到的三个文件时间：
    1. 修改时间（modification time，mtime）：当文件的内容数据更新时，就会更新这个时间；
    2. 状态时间（status time，ctime）：当文件的状态（status）发生改变时，就会更新这个时间；
    3. 读取时间（access time，atime）：当文件的内容被读取时，就会更新这个时间；
  • touch [-acdmt] file
    • a：仅自定义access time；
    • c：仅修改文件的时间，若文件不存在则不建立新文件；
    • m：仅修改mtime；

执行touch -t 201406150202 1.txt,可以发现该文件的mtime和atime都改变了，但是ctime没有改变；我们在复制文件的时候可以复制文件的所有属性，但是也无法复制文件的ctime这个属性；

  • touch这个名利最常用来：
    • 建立一个空的文件；
    • 将某个文件日期定义为目前（mtime与atime）；

文件的默认权限：umask

  • 可以通过umask查看文件的默认权限

    • umask(0022);umask -S(u=rwx,g=rx,o=rx)
    • 默认的文件最大权限为：666（没有可执行的权限）；目录权限为：777；
    • 建立文件是（-rw-rw-rw-）-(—–w–w-)=(-rw-r–r–);(666)-(022)=(644)
    • 建立目录时（drwxrwxrwx）-(—–w–w-)=(-rwx-r-xr-x);(777)-(022)=(755)

文件的特殊权限：SUID、SGID、SBIT

  • setuid(SUID): 针对命令和二进制程序的，当普通用户执行某个(passwd)命令的时候，可以拥有这个命令对应用户的权限,即让普通用户可以以root用户的角色执行程序或命令。
  • setgid(SGID)：希望一个目录被多个用户(同属于一个组)共享，同一个组的用户可以处理
  • stick bit(SBIT) (粘贴位)就是：除非目录的属主和root用户有权限删除它，除此之外其它用户不能删除和修改这个目录
